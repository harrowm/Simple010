# rosco_m68k r2 CPLD — ATF1508AS PLCC84

Replaces IC2 (address decoder), IC3 (DUART select), IC5 (glue logic),
IC6 (watchdog), IC4 (XR68C681 UART A, fixed 115200 8N1), and IC7 (74LS148
interrupt priority encoder) with a single ATF1508AS CPLD.

## Build

Requires [atf15xx_yosys](https://github.com/hoglet67/atf15xx_yosys.git) cloned
into `~/atf15xx_yosys`, and `iverilog`/`vvp` on PATH.

```
./build.sh
```

Runs synthesis, fits the design, verifies all 56 pin preassignments are
respected, then runs the testbench. Expected output:

```
Synthesising... done.
Fitting... done.  Fitter completed — 122/128 MCs used.
Checking pin assignments... all 56 assignments respected.
Running testbench... 120/120 tests passed.
```

On failure the script exits non-zero and prints the specific error (mismatched
pins or failing test assertions).

## Testing only

To run the testbench without re-synthesising:

```
iverilog -DSIMULATION -o tb_cpld tb_cpld.v cpld.v && vvp tb_cpld
```

The `-DSIMULATION` flag is required — without it only 118/120 tests pass.
The two affected tests exercise the RBA-read/cpu_rd_rba handshake: in synthesis
this is a 1-cycle SYS_CLK pulse (~124 ns) that can be missed by UART_CLK
(~270 ns period); the simulation path holds the signal until UART_CLK acks.

## Files

| File | Description |
|------|-------------|
| `cpld.v` | RTL source and `//PIN:` preassignment comments |
| `tb_cpld.v` | Testbench (120 assertions) |
| `build.sh` | Synthesis + fit + pin check + simulation |
| `cpld.jed` | Fitter output — program this into the device |
| `cpld.fit` | Fitter log (resource usage, pin placement) |
| `cpld.pin` | Pin placement file (generated by fitter) |
